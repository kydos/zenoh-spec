= Authentication
:description: Authentication mechanisms available during Zenoh session establishment.

Authentication verifies the identity of a node before a session is allowed to proceed.
Zenoh supports both transport-layer and session-layer authentication.

== Transport-Layer Authentication (TLS / QUIC)

When a TLS or QUIC link is used, mutual TLS (mTLS) provides strong authentication via X.509 certificates.

* The server certificate is always validated by the client.
* The client certificate is optional (`ext_auth` not required) but RECOMMENDED for production deployments.
* The Subject or Subject Alternative Name (SAN) of the peer certificate is available to the access control layer.

== Combining Mechanisms

Transport-layer and session-layer authentication may be used simultaneously.
Access control decisions MAY use identity from either layer.
